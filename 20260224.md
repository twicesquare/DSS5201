## 时间按钮
过滤/放大特定的时间片

![](https://cdn.jsdelivr.net/gh/twicesquare/pics@main/20260224211010424.png)

| 参数名 | 作用（中文解释） | 可选值示例 / 说明 |
| :--- | :--- | :--- |
| label | 按钮文字 | 出现在图表按钮上的文本，如 "1月", "1年", "全部"。 |
| count | 步进数量 | 点击按钮后，时间轴移动/缩放的单位数量（整数）。 |
| step | 时间单位 | 指定移动的时间周期，可选：'month' (月), 'year' (年), 'day' (天), 'hour' (时), 'all' (全部)。 |
| stepmode | 移动模式 | 决定时间范围如何计算，主要有 'backward' 和 'todate' 两种。 |

```
# 示例：创建一个显示“过去 6 个月”的按钮
dict(count=6, label="6个月", step="month", stepmode="backward")
```

```
import plotly.express as px
import pandas as pd

# 假设你有一份时间序列数据
# fig = px.line(df, x='date_column', y='value_column')

fig.update_xaxes(
    rangeselector=dict(
        buttons=list([
            # 示例 1: 使用 'backward' 模式查看过去一个月
            dict(count=1, label="1个月", step="month", stepmode="backward"),
            
            # 示例 2: 使用 'todate' 模式查看本年初至今
            dict(count=1, label="今年至今", step="year", stepmode="todate"),
            
            # 示例 3: 常用“全部”按钮
            dict(step="all", label="全部")
        ])
    )
)
```

### 控制方法 `methods`
| 方法 (Method) | 作用描述 |
| :--- | :--- |
| restyle | 修改数据属性。例如：点击按钮切换线条颜色、改变图表类型（如从折线图变柱状图）。 |
| relayout | 修改布局属性。例如：改变标题、修改坐标轴的范围或刻度（Linear vs Log）。 |
| update | 同时修改数据和布局。是最全能但也最复杂的控制方式。 |

### `args`
args 列表，里面存放的是传递给上述 method 的具体指令。

如果你想切换数据列，args 里就会写上目标列的名称。

如果你想切换坐标轴，args 里就会写上 {"xaxis.type": "log"}。

## slider 滑块
通过移动滑块更新图表
![](https://cdn.jsdelivr.net/gh/twicesquare/pics@main/20260224223101987.png)

**`animation_frame`:**

这个参数定义了动画的帧（Frames）。

作用：指定数据集中的一列（通常是年份、月份或步骤），Plotly 会根据这一列的每一个唯一值生成一个动画帧。

效果：图表下方会自动出现一个播放/暂停按钮以及一个滑动条（Slider），用户可以拖动滑块查看不同时期的状态。

示例：如果设置 animation_frame="year"，图表就会按照年份顺序播放。

**`animation_group`:**

这个参数用于确保动画在切换帧时，点与点之间的对应关系是正确的。

作用：提供一个持久的标识符，告诉 Plotly 哪些数据点在不同帧之间属于同一个实体。

为什么重要：如果没有设置，当动画播放时，点可能会出现杂乱的闪烁或位移。设置后，Plotly 可以平滑地计算同一个对象从位置 A 移动到位置 B 的轨迹。

示例：在展示各国人均 GDP 随时间变化的散点图中，animation_group="country" 确保了代表“中国”的那个圆点能够连贯地从 2020 年移动到 2021 年。

### `args`的应用1
```
# Create the buttons
my_buttons = [
    {'label': "Bar plot", 'method': "update", 'args': [{"type": "bar"}]},
    {'label': "scatterplot", 'method': "update", 'args': [{"type": "scatter", 'mode': 'markers'}]}
]
```

这里定义了一个包含两个字典的列表，每个字典代表一个按钮：

label：按钮上显示的文本。

method: "update"：这是最强大的方法，表示同时修改图表的数据（如类型）和布局（如标题）。

args：传递给图表的参数指令。

第一个按钮将图表类型强制改为 "bar"。

第二个按钮将类型改为 "scatter"，并指定模式为 'markers'（即纯散点）。

### 注释（Annotations）
```
value_annotations=[{'text': 'Sept was the best', 'showarrow': True, 'x': 'September', 'y': 345397 }]
volume_annotations=[{'text': 'Oct was the best', 'showarrow': True, 'x': 'October', 'y': 71900 }]
```
这里定义了两个列表，分别存储不同数据维度下的文字说明。

showarrow: 设置为 True 表示会画一个箭头指向具体的坐标点 (x, y)。

### `args`的应用2
```
my_buttons = [
{'label': "By Sales Value", 'method': "update", 'args': [{"visible": [False, True]}, {"annotations": value_annotations}]},
{'label': "By Sales Volume", 'method': "update", 'args': [{"visible": [True, False]}, {"annotations": volume_annotations}]}]
```
method: "update": 表示点击时同时修改 数据(Trace) 和 布局(Layout)。

args 的第一个字典 `{"visible": [...]}`: 控制哪一组数据可见。

`[False, True]` 表示隐藏第一个 Trace（Volume），显示第二个 Trace（Value）。

`args` 的第二个字典 `{"annotations": ...}:` 修改全局布局中的注释，确保显示的文字内容与当前看到的数据相匹配。


### 按钮的集成
```
# Add buttons to the plot and show
fig.update_layout({
    'updatemenus': [{
      'type': "buttons",        # 指定交互界面为按钮（也可以是 "dropdown" 下拉框）
      'direction': 'down',      # 多个按钮的排列方向为“向下”
      'x': 1.3, 'y': 0.5,       # 按钮菜单在画布上的位置（1.3 表示位于右侧画布边缘外）
      'showactive': True,       # 点击后按钮保持高亮状态
      'active': 0,              # 默认激活索引为 0 的按钮（即第一个柱状图按钮）
      'buttons': my_buttons}]   # 将前面定义的 my_buttons 列表存入
})
```
